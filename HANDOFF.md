# Handoff: js-deworder

The static browser app has a working cleaner pipeline, upload/paste mapping
flow, preview, copy, download, and browser-native tests.

## Current state

- `PRD.md` contains the product requirements and cleaner spec.
- `index.html` and `static/style.css` are the browser-only UI.
- `static/defaults.js` holds default mappings, allowed targets, disallowed
  tags, stripped attrs, and storage keys.
- `static/deworder.js` implements `decodeHtmlBytes(arrayBuffer)`,
  `detectClasses(htmlString)`, and `clean(htmlString, config)`.
- `index.html` uses plain ordered `<script>` tags instead of ES modules so the
  app can run when opened directly as `file://.../index.html`.
- Preview heading navigation is wired: `prev-heading-btn` / `next-heading-btn`
  share `state.headingIndex` and step both panes through `<h1>`–`<h6>`. The
  before pane also matches any classes the user mapped to `h1`–`h6`.
  Buttons enable on the after iframe's `load` event.
- The three step-indicator chips (`ladda upp` / `mappa` / `förhandsgranska`)
  are click + keyboard activatable for jumping between steps.
- `clean()` runs pre-parse cleanup, body parsing, namespace/disallowed tag
  removal, `SpellE`/`GramE` unwrap, comment class stripping, `<p>`/`<span>`
  class mapping, comment-style removal, `<div>` unwrap, `<li>` run wrapping
  into `<ul>` or `<ol>` (classes mapped to `ol` produce `<ol>` wrappers;
  runs split on type change), inline tag normalization, link URL preservation,
  `<br>` removal, attribute stripping, optional class stripping, empty span
  unwrap, nested `strong`/`em` collapse, empty element removal, and
  body-fragment serialization.
- `static/app.js` wires the intended UI flow. Upload/paste can advance through
  mapping and preview; copy/download controls are connected to the cleaned
  fragment.
- Drag/drop stores the dropped `File` in app state instead of assigning to
  `input.files`, which is unreliable across browsers. File picker upload still
  uses the native file input.
- `tests/fixtures/verify-fixture.html` is the small stable verifier fixture.
- `tests/fixtures/verify-fixture.cleaned.html` is the checked-in expected
  output for exact-string parity on that fixture.
- `tests/deworder.test.html` is a browser-native test page for decoding,
  class detection, exact verifier-fixture output, and focused cleaner
  behavior.
- `tests/production-smoke.html` runs the cleaner against the real production
  fixture through the byte-decoding path and checks for forbidden Word/browser
  artifacts.
- `tests/Buddhismen LL 250905.html` is a real production Word export fixture.

## Decisions already made

- `blockquote` is an allowed mapping target.
- Keep `NoKArbetsinfo -> p` and `NoKBetoningFetFrg -> p`.
- Keep cleaned output as a body fragment, not a full HTML document.
- Store mappings globally in `localStorage`; no preset export/import in v1.
- Support file upload as the primary path and paste/copy as a convenience
  path.
- Keep preview iframes as `sandbox="allow-same-origin"` so heading navigation
  can inspect and scroll both panes. Do not add `allow-scripts`,
  `allow-forms`, or `allow-top-navigation`.
- Heading navigation between headings in preview is low priority. If it
  complicates preview work, put it on ice and prioritize cleaner correctness,
  rendered before/after panes, source copy, and download.

## Deferred ideas

- **Class injection after mapping.** When a class like `NoKRubrik1` maps to
  `h1`, the original class is stripped. A future option could preserve the
  source class on the output element (e.g., `<h1 class="NoKRubrik1">`) or
  let users specify a custom output class per mapping row (e.g.,
  `<h1 class="section-heading">`). The source class is still present on the
  element at the moment of replacement, so no extra detection pass is needed —
  it's a UI + pipeline change. This is most useful when cleaned output lands
  in a CMS with its own stylesheet.

## Production fixture notes

- `tests/Buddhismen LL 250905.html` is about 311 KB and declares UTF-8 via
  `<meta http-equiv=Content-Type content="text/html; charset=utf-8">`.
- The file contains Word/Office namespace blocks, VML/image fallbacks, a Word
  comment list, comment anchors, JavaScript event attributes, `link`/`meta`
  head clutter, `script`, `img`, and `hr` tags. These make it a good fixture
  for ongoing cleaner work.
- Observed frequent classes include `MsoNormal`, `NoKText`, `NoKPunktlista`,
  `NoKBildtext`, `NoKTextindrag0`, `NoKRubrik3`, `MsoCommentReference`,
  `msocomoff`, `msocomtxt`, and `msocomanchor`.
- `NoKTextindrag0` and its `CxSpFirst`/`CxSpMiddle`/`CxSpLast` variants are
  not in the default map. That is acceptable: detected classes appear in the
  mapping table so the user can choose targets per document/template.
- The fixture includes normal `NoKCitatCxSp*` variants; class normalization
  exposes them as `NoKCitat`.
- JS production smoke currently succeeds on this fixture:
  - raw length: 309,014 chars;
  - cleaned length: 33,756 chars;
  - detected class rows: 14;
  - heading counts in cleaned output: 1 `<h1>`, 7 `<h2>`;
  - no forbidden hits for `<script`, `<img`, `<style`, `<meta`, `mso-`,
    `MsoComment`, `msocom`, `onclick`, `<o:`, `<w:`, or `<!--`.

  `CxSp*` normalization, Word comment filtering, `windows-1252` decoding, and

- A large paste-mode app smoke with the production fixture reached preview,
  produced 14 mapping rows, rendered cleaned source beginning with
  `<h1>Buddhism</h1>`, preserved Swedish text, copied the cleaned source to
  the clipboard, showed copy/download success labels, and cleared stale
  preview/source state on reset. It reported 301,579 pasted characters ->
  33,756 cleaned characters.
- Manual file upload with `tests/Buddhismen LL 250905.html` was tried by
  David and works fine.

  files into `input.files`; `static/app.js` now keeps the dropped file
  separately in state. This still needs manual confirmation in a real
  browser.

  do not reliably load from `file://`; scripts are now plain globals loaded
  in order. The in-app browser blocks `file://` navigation, so this still
  needs manual confirmation by opening the file directly.
- Local verification was run from a static server in `js-deworder`. Port
  `8765` may be occupied, in which case `python3 -m http.server 8766` and
  `http://localhost:8766/` is the fallback.
